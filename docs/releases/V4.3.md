# Release V4.3 - Launch Experience Release

## Executive Summary

V4.3 improved the local launch experience so the product feels like local software rather than a developer server.

## Objectives

- Add a product landing page.
- Make `/workspace` the primary Decision Workspace route.
- Preserve `/dashboard/` backward compatibility.
- Introduce product-facing launch commands.
- Keep Decision Intelligence behavior unchanged.

## Deliverables

- Landing page at `/`.
- Decision Workspace route at `/workspace`.
- Backward-compatible `/dashboard/` route.
- `launch.py` and `run_app.sh`.
- Updated README and local setup docs.

## Non-goals

- No dashboard redesign.
- No Decision Brief changes.
- No evidence, readiness, pathway, review, or evaluation changes.
- No installer, Docker, Electron, database, authentication, or cloud deployment.

## Validation

- `.venv/bin/ruff check .`
- `python3 -m py_compile app.py src/*.py launch.py`
- `python3 -m pytest`
- `node --check dashboard/app.js`
- `node --check dashboard/project.js`

## Known Issues

- V4.3 initially exposed an asset-loading issue when `/workspace` served the dashboard HTML with relative CSS/JS paths. V4.4 resolved this through absolute dashboard asset paths.

## Next Release Preview

V4.4 focuses on double-click local launchers for non-technical users.
