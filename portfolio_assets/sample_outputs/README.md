# Sample Outputs

Store curated output examples for portfolio review.

Do not hand-edit generated artifacts here unless the file clearly states it is a curated excerpt. Generated run artifacts belong under `outputs/` or `demo_case_outputs/`.

Start with the V4 workspace demo:

```text

```

## What Reviewers Should Notice

- Project artifacts are local and inspectable.
- Evidence library items are distinct from generated analysis.
- Multiple questions can be linked to one project.
- Decision history and delta support traceability over time.
- Markdown and JSON outputs serve different reviewer needs.
