# Release V4.6 - Portfolio Demo Package Release

## Executive Summary

V4.6 adds a professional demonstration package so reviewers can understand the product without running it.

## Objectives

- Provide a three-minute product review path.
- Make demo artifacts easier to inspect.
- Explain what reviewers should observe in the V4 workspace demo.
- Separate curated portfolio assets from generated outputs.
- Preserve all product behavior.

## Deliverables

- `docs/DemoWalkthrough.md`
- `docs/PortfolioReviewGuide.md`
- `portfolio_assets/demo_notes/`
- Expanded guidance in `portfolio_assets/screenshots/`, `sample_inputs/`, and `sample_outputs/`
- README demo links
- Documentation Index demo and portfolio review links

## Non-goals

- No new product capabilities.
- No Decision Intelligence logic changes.
- No evidence, retrieval, routing, API, evaluation, or reviewer workflow changes.
- No generated output changes.
- No dashboard redesign.

## Validation

V4.6 should pass:

- Markdown link check
- `.venv/bin/ruff check .`
- `python3 -m py_compile app.py src/*.py launch.py`
- `python3 -m pytest`
- `node --check dashboard/app.js`
- `node --check dashboard/project.js`

## Known Issues

- Portfolio screenshots remain placeholders until a populated Decision Workspace screenshot is captured.
- Demo artifacts are static examples and should not be described as live retrieval or autonomous analysis.

## Next Release Preview

Future releases should replace placeholder screenshots with populated workspace visuals and continue improving reviewer trust without expanding product scope.
