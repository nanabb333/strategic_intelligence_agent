# Release V4.4 - Product Distribution Release

## Executive Summary

V4.4 added lightweight double-click launchers for local product distribution.

## Objectives

- Let non-technical reviewers launch the product without VS Code.
- Keep the product local-first and lightweight.
- Preserve developer startup commands.
- Avoid installer, desktop-app, cloud, or packaging complexity.

## Deliverables

- `launch/Launch Strategic Intelligence Decision Companion.command`
- `launch/Launch Strategic Intelligence Decision Companion.bat`
- `launch/README.md`
- Shared launcher improvements in `launch.py`.
- Documentation updates for double-click startup.
- Absolute dashboard asset paths so `/workspace` and `/dashboard/` both load styled assets.

## Non-goals

- No Electron, Tauri, Docker, database, authentication, auto-update system, or installer framework.
- No changes to Decision Intelligence, evidence processing, retrieval, evaluation, or reviewer workflow.
- No generated output changes.

## Validation

- `.venv/bin/ruff check .`
- `python3 -m py_compile app.py src/*.py launch.py`
- `python3 -m pytest`
- `node --check dashboard/app.js`
- `node --check dashboard/project.js`
- Local route checks for `/`, `/workspace`, `/dashboard/`, `/health`, and dashboard assets.

## Known Issues

- Users still need Python 3 installed.
- macOS security settings may require approval before running a downloaded `.command` file.
- This release does not produce a signed `.app` or Windows installer.

## Next Release Preview

V4.5 focuses on enterprise repository presentation, documentation architecture, release notes, and portfolio-readiness structure.
