# Repo 5 Decision Philosophy

Repo 5 is not designed to predict the future.

It is designed to support human decision quality under uncertainty. That objective requires future human evaluation and is not an automated product score.

## Core Principles

### 1. History is evidence, not the answer

Historical cases should support reasoning.

They should not replace reasoning.

---

### 2. The product should answer Why, So What, and Now What

Every output should help the user understand:

- Why this matters
- So what it means for the decision
- Now what should be done next

---

### 3. An assessment must be revisable

No assessment or reviewer selection should be treated as permanent.

Every assessment should expose:

- assumptions
- trade-offs
- what evidence could change the answer

---

### 4. Decision quality matters more than prediction accuracy

The product should not claim certainty or that it has measured decision quality.

It should help users make better choices even when the future is uncertain.

---

### 5. The system should reduce decision uncertainty

Every module should answer:

Does this help the user make a clearer, better, more informed decision?

If not, it should not be added.
