# Repo Terminology

This document defines the core terminology used throughout Repo 5.

These definitions are intended to ensure consistent language across the architecture, documentation, source code, and user-facing outputs.

---

## Change Triggers

### Definition

New evidence or changing conditions that could alter the current recommendation.

### Purpose

Ensures that recommendations remain adaptive rather than static.

---

## Decision Criteria

### Definition

The explicit factors used to evaluate and compare decision paths.

### Purpose

Ensures that all options are assessed using transparent and consistent standards.

---

## Decision Paths

### Definition

The realistic courses of action available to the decision-maker.

### Purpose

Defines the feasible alternatives before selecting a preferred path.

---

## Decision Question

### Definition

The primary decision that the user is attempting to make.

### Purpose

Focuses the analysis on the actual decision rather than the surrounding event.

---

## Historical Evidence

### Definition

Historical cases and observed outcomes that support the reasoning process.

### Purpose

Provides supporting evidence without allowing historical analogues to become the decision itself.

---

## Preferred Path

### Definition

The decision path that is currently best supported by the available evidence and reasoning.

### Purpose

Represents the system's current judgment while remaining open to revision if new evidence emerges.

---

## Trade-offs

### Definition

The benefits gained, costs accepted, opportunities sacrificed, and remaining uncertainty associated with each decision path.

### Purpose

Makes the consequences of each option explicit instead of presenting recommendations without context.
